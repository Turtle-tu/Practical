# Техническое руководство: Разработка системы автоматизации определения объёма груза по данным камер видеонаблюдения

## Цель документа

Пошаговое руководство по разработке программной системы автоматизированного определения объёма груза в зоне приёмки склада. Документ описывает исследование предметной области, архитектуру системы, реализацию алгоритмов компьютерного зрения, процесс контейнеризации и подготовку итогового MVP-продукта для передачи заказчику.

## 1. Исследование предметной области

### Ключевые задачи проекта

Проект направлен на автоматизацию процесса оценки объёма грузов на складе с использованием изображений и видеопотока с камер видеонаблюдения.

Система должна:

- Определять паллету и груз на изображении
- Выполнять сегментацию или детекцию области груза
- Оценивать высоту груза
- Рассчитывать примерный объём
- Отображать результат в понятном интерфейсе
- Поддерживать работу через веб-интерфейс
- Быть готовой к развёртыванию в инфраструктуре заказчика

### Почему выбран Python

- Большое количество библиотек для компьютерного зрения и машинного обучения
- Простая интеграция с OpenCV, NumPy и веб-фреймворками
- Удобство прототипирования алгоритмов
- Поддержка Docker и серверного развёртывания
- Подходит для исследовательских и MVP-проектов

### Почему используется Docker

- Позволяет передать заказчику полностью готовое окружение
- Исключает необходимость ручной установки зависимостей
- Упрощает перенос между серверами
- Позволяет запускать систему одной командой
- Изолирует систему от инфраструктуры заказчика

### Анализ ограничений предметной области

В процессе разработки были выявлены следующие особенности:

- Камеры склада работают с перспективными искажениями
- Качество изображения зависит от освещения и погодных условий
- Груз может иметь сложную форму
- Заказчик пока не готов к полноценному внедрению системы в производственный контур
- Доступ к инфраструктуре заказчика отсутствует

**Поэтому итоговым артефактом проекта стал контейнеризированный MVP-продукт.**

## 4.3 Оценка высоты груза

В проекте использовались два подхода.

### Упрощённый метод

Высота определяется через размеры bounding box.

**Формула:**

```
h = h_px ⋅ (h_cam / f_x)
```


**Где:**
- `h_px` — высота объекта в пикселях;
- `h_cam` — высота камеры;
- `f_x` — фокусное расстояние.

### Метод 3D-проекции

Была разработана более точная математическая модель оценки высоты.

**Использовались:**
- положение камеры;
- PnP-решение;
- геометрия лучей;
- пересечение с плоскостью поддона.

### Пример функции

```python
estimate_height_3d(cargo_mask, best_hfov)
```

### Логика работы

1. Нижние пиксели маски интерпретируются как основание груза.
2. Верхние пиксели определяют направление лучей.
3. Выполняется least-squares оценка высоты.
4. Рассчитывается итоговая высота объекта.

### Ограничения реализации

На текущем этапе основным рабочим методом остаётся оценка через 2D bounding box.

Полноценная 3D-реконструкция требует:

- калибровки камеры;
- дополнительных тестовых данных;
- точного определения параметров съёмки.

## 4.4 Расчёт объёма

После определения площади основания и высоты вычисляется объём груза.

**Формула:**

```
V = S ⋅ h
```

**Где:**
- `V` — объём;
- `S` — площадь основания;
- `h` — высота.

### Пример вычисления

```python
volume = base_area * height
```

### Сглаживание результатов

Для видеорежима используется скользящее среднее.

```python
SMOOTH_WINDOW = 5

```

Это уменьшает скачки значений между соседними кадрами.
