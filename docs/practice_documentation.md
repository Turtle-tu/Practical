# Цели и задачи практики

## Цель

Разработка автоматизированной системы измерения объёма грузов с использованием технологий компьютерного зрения и искусственного интеллекта.

---

## Задачи

- Изучить процессы складской логистики и методы автоматизированного измерения грузов;
- Исследовать существующие решения и технологии компьютерного зрения;
- Реализовать алгоритм обнаружения грузов на изображениях;
- Разработать функцию вычисления объёма грузов на основе полученных данных;
- Реализовать пакетную обработку изображений и экспорт результатов в CSV;
- Подготовить Docker-инфраструктуру для быстрого развёртывания проекта;
- Провести тестирование системы и анализ точности вычислений;
- Подготовить полный комплект технической и отчётной документации.

---

# Используемые технологии

- Язык: Python 3.10+
- Компьютерное зрение: OpenCV, YOLO
- Контейнеризация: Docker, Docker Compose
- Форматы данных: CSV, JSON, UTF-8
- Инструменты: Git, GitHub, VS Code
- Обработка данных: NumPy, Pandas
# Основной функционал проекта

- Автоматизированная система измерения объёма грузов на основе компьютерного зрения.
- Анализ изображений и видео с камер наблюдения для поиска паллет и грузов.
- Использование моделей YOLO для сегментации и обнаружения объектов.
- Расчёт физических параметров и объёма грузов.
- Поддержка обработки фото и видеопотока.
- Подготовка веб-интерфейса для загрузки изображений и отображения результатов анализа.
- Контейнеризация проекта с помощью Docker для быстрого развёртывания.
- Формирование JSON-результатов для интеграции с WMS/TMS.
- Проведение экономического анализа и расчёта unit-экономики проекта.
- Подготовка аналитических материалов: конкурентный анализ, диаграммы, архитектурные схемы.
- Поддержка модульной архитектуры для дальнейшего расширения функционала.

#  Этапы выполнения практики

## Анализ требований (1–2 недели)

Изучение процессов работы транспортной компании и складской логистики. Проведение обсуждений с представителями организации-партнёра, определение проблем ручного измерения объёма грузов и формирование технического задания.

## Проектирование системы (3 неделя)

Улучшение архитектуры решения, выбор технологий компьютерного зрения и моделей YOLO. Проектирование пайплайна обработки изображений и схемы интеграции с WMS/TMS.

## Разработка ядра системы (4–5 недели)

Реализация алгоритмов обнаружения паллет и грузов, обработка масок сегментации, вычисление размеров объектов и расчёт объёма.

## Интеграция и веб-интерфейс (6 неделя)

Разработка REST API и веб-интерфейса для демонстрации работы системы. Подготовка Docker-контейнера для удобного развёртывания проекта.

## Тестирование и оптимизация (7–8 недели)

Проверка работы алгоритмов на тестовых изображениях и видео, анализ точности измерений, устранение ошибок обработки и оптимизация производительности.

## Документирование и сдача проекта (9 неделя)

Подготовка итоговой документации, презентации, расчётов unit-экономики и демонстрационных материалов для заказчика и преподавателей.

#  Взаимодействие с организацией-партнёром

Практика проходила совместно с организацией-партнёром, заинтересованной в автоматизации процессов складской логистики и измерения объёма грузов.

Взаимодействие включало:

- Демонстрацию промежуточных версий алгоритмов и интерфейса.
- Совместное обсуждение архитектуры внедрения через Docker-контейнеры.
- Анализ ограничений внедрения в инфраструктуру заказчика.

Отдельное внимание уделялось вопросам информационной безопасности и невозможности прямого доступа к инфраструктуре компании, поэтому итоговым решением стало контейнеризированное развёртывание системы.

#  Экономическая эффективность проекта

Согласно выполненным расчётам:

- Экономия времени обработки одного груза: ~2,5 минуты.
- Снижение затрат на обработку одного груза: ~17 руб.
- Дополнительный эффект от уменьшения ошибок и рекламаций: ~5 руб.
- Совокупный эффект на один груз: ~26 руб.
- Операционные затраты системы: ~5 руб./груз.
- Чистый эффект (Unit Profit): ~21 руб./груз.
- Ожидаемый экономический эффект: ~420 000 руб./месяц при обработке 20 000 грузов.

Проект продемонстрировал высокую рентабельность и перспективность внедрения в складскую логистику.
